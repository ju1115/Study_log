# 🛠️ `Next.js_Prettier_convention`

## 🎯 Goal (목표)

> **"저장(Cmd+S) 한 번으로 코드 스타일 통일하기"**

- 혼자 개발하더라도, 혹은 팀원과 함께하더라도 항상 일관된 코드 스타일(들여쓰기, 따옴표 등)을 유지.
- VS Code 설정을 프로젝트 내부에 저장하여, 팀원 모두가 별도 설정 없이 동일한 개발 환경 경험.
- Tailwind CSS 클래스 순서를 자동으로 정렬하여 가독성 확보.

## 💻 Implementation (구현 과정)

### Step 1. Setup (설치 및 확장 프로그램)

**1. VS Code Extension 설치 (필수)**

- `Prettier - Code formatter`
- `ESLint`

**2. Tailwind CSS 정렬 플러그인 설치 (Terminal)**

```bash
npm install -D prettier-plugin-tailwindcss
```

### Step 2. Code Snippet (설정 파일 작성)

**1. Prettier 설정 파일 (`.prettierrc`)**
프로젝트 최상위 루트에 생성합니다. 현업에서 가장 많이 사용하는 '국룰' 옵션입니다.

```json
{
  "semi": true,
  "singleQuote": true,
  "tabWidth": 2,
  "trailingComma": "all",
  "printWidth": 100,
  "bracketSpacing": true,
  "arrowParens": "always",
  "plugins": ["prettier-plugin-tailwindcss"]
}
```

**2. VS Code 워크스페이스 설정 (`.vscode/settings.json`)**
프로젝트 루트에 `.vscode` 폴더를 만들고 그 안에 파일을 생성합니다.
이 설정 덕분에 **"저장할 때(On Save)"** 자동으로 포맷팅이 실행됩니다.

```json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": "explicit"
  },
  "files.eol": "\n"
}
```

## 💡 Best Practices

- **프로젝트 단위 설정 공유 (.vscode):**

  - 현업에서는 개인의 VS Code 설정(User Settings)보다, 프로젝트 폴더 안의 `.vscode/settings.json`을 사용하여 팀원 간의 실행 환경을 강제로 통일합니다. (누구는 저장이 되고, 누구는 안 되는 상황 방지)

- **Tailwind Class Sorting:**

  - `prettier-plugin-tailwindcss`를 사용하면 `class="p-4 flex bg-white"` 처럼 뒤죽박죽인 클래스들을 `class="flex bg-white p-4"` 처럼 **Tailwind 권장 순서(Layout -> Box Model -> Visual...)**대로 자동 정렬해 줍니다. 이는 유지보수에 매우 큰 도움이 됩니다.

- **주의할 점 (Gotchas):**
  - Next.js 프로젝트 생성 시 ESLint가 이미 설치되어 있습니다. Prettier와 ESLint가 충돌하지 않도록, 포맷팅(스타일)은 Prettier에게, 코드 문법 검사는 ESLint에게 확실히 역할을 분담시키는 것이 중요합니다. (위 설정대로면 충돌 없이 잘 작동합니다.)
